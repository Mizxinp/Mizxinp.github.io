今天在使用ant design mobile的Grid组件遇到了两个问题

1.组件存在的bug

当设置了grid属性是四行九列，但一开始只出现一行，当滑动后才显示正常。

官方给出的解决方案是派发一个事件：

这样就可以解决这个问题了

2.

滑动时报错，功能能够正常运行，chrome下报错

大概意思是：不能给passive(被动的)事件监听器preventDefault，因为它被认为是passive。

火狐下给出警告忽略一个注册为 ‘passive’ 的监听器的事件类型为 ‘touchmove’ 的 ‘preventDefault()’ 调用

这里有一篇类似的问题：关于passive event listener的一次踩坑

最后在 Chrome Dev Tools - Mobile 这篇文章中得到解决。

原文是这样描述的：

Sorry for the trouble, this is a breaking change in Chrome 56 to improve scroll performance. You probably need to add an appropriate touch-action CSS rule to explicitly disable touch scrolling

主要是使用了css属性touch-action,将其设为none即可解决问题。具体可以参考MDN有关touch-action的介绍
